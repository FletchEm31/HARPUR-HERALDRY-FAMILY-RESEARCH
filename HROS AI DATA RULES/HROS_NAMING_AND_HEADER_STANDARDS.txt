SECTION 3 — FILE NAMING CONVENTION
================================================================

  FORMAT:   HROS_[CATEGORY][NUMBER]_[DESCRIPTOR].ext

  CATEGORIES:
    BOOT      Boot protocol files — loaded at session start
    PROTO     Protocol and rules files
    SOURCE    AI source data files (the research content)

  RULES:
    - All caps for the HROS prefix and category
    - Number is zero-padded where sequence matters (e.g. BOOT2, SOURCE11)
    - Descriptor uses underscores, no spaces, all caps
    - No version numbers in file names — version lives in header only
    - File names never change after creation
      (archived versions may have _ARCHIVE or date suffix)

  EXAMPLES:
    HROS_BOOT2_SESSION_INITIALIZATION_RULE_MANDATORY.txt
    HROS_PROTO_RESEARCH_DISCIPLINE_RULES.txt
    HROS_SOURCE1_ANNEX_A_VERIFIED_SUPPORTING_EVIDENCE.md
    HROS_SOURCE8_DOCUMENT_ANALYSIS_REGISTER.md

  FILE EXTENSIONS:
    .txt    Protocol, boot, and rules files (HROS AI DATA RULES folder)
    .md     Source data and research content files (HROS AI SOURCE DATA folder)

SECTION 6 — VERSION CONTROL
================================================================

  - Version number lives in the header ONLY
  - File names never include version numbers
  - Version increments:
      Minor edit (content addition, correction):   X.0 → X.1
      Major revision (restructure, new section):   X.1 → Y.0
  - Updated: field must be changed every time the file is saved
  - Archived versions may be renamed with suffix _ARCHIVE_[YYYYMMDD]
    and stored outside the active GitHub repository
